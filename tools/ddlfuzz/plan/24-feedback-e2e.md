# 24 — bounded feedback + e2e-gen dedup

Merge of the former 24 (e2e-gen dedup) and 25 (bounded feedback) drafts — one worktree
(`worktrees/feedback-e2e`), one doc, staged landings. The two parts touch disjoint packages
(Part I: `internal/compare`, `cmd/ddlfuzz`, `internal/corpus`, `internal/oracle`, `oracle/`;
Part II: `internal/gen`, `e2e/`) and are independently correct; §Sequencing binds the landing
order and the two watch-window interactions.

## Part I — bounded feedback: fixed-size feature maps, per-case edge attribution

Plan 23 built the right signal (behavior novelty + oracle edges) on the wrong substrate: an
unbounded string key with a cap bolted on. Rebuild it the way coverage-guided fuzzers do —
features live in a fixed-size space by construction, so novelty tapers instead of exploding,
saturation is a stat instead of a failure mode, and the per-exec cost is a hash and a bit test.

### Diagnosis (2026-07-04, hours after plan 23 landed)

Plan 23's two dedup premises failed on contact:

- *"steady-state cost is a map probe"* (23 §1): `BehaviorRawHash` hashes identifier-bearing raw
  sig/error text, so the first-level seen-set never hits — every exec pays the full shaped-key
  build (~4.1µs, 70 allocs, benched) plus two `es.mu` hops.
- *"cap 1M keys; when full, stop"*: both engines hit the cap in minutes; `seen_set_full` reached
  28M by 96M execs — ~65% of cases still produce a never-seen key *after* the cap. The key space
  isn't behavior classes, it's near-injective over inputs. The full-check runs after the key is
  built, so "stop inserting" never stopped the paying.
- Consequences: retention storm (SQLite insert + pool eviction per case) crushed throughput to
  ~10K/s; once the caps froze retention, steady state recovered to only ~250K/s (residual key
  cost) — masked in stats because `execs_per_sec` is a lifetime average (`fuzzloop.go:826`).
  corpus.db: 22.4GB / 2.2M rows, `retention_skipped=0` (budget never braked). Feedback now
  inert: all cost, zero signal. Sets are in-memory, so every restart replays the storm.

Where the key diversity comes from (offline probe over 500K saved rows, re-parsed;
`worktrees/keyspace-probe/.../compare/keyspace_probe_test.go`):

- `behavior` rows (1.74M): **structural** `sigShape` diversity — type families × spec
  kind/order/count × multi-statement combos — doubled by the oracle-side `digestShape` running
  the same machinery. Quote-masking collapses 0%; masking all digits only −6.7%. `modeMask` is
  a saturated ×16, harmless.
- `noise` rows (418K): our parser's errors are fully templated (34 distinct shapes in 200K;
  `%q` and `at byte N` already masked). All four measurable key fields collapse to 852 combos —
  the diversity lives in the one unmeasured field, `digestShape = NormalizeError(d.Error)`:
  `NormalizeError` masks only double quotes (`compare.go:751`), while the servers quote
  fragments in single quotes and emit `at line N`.

Counterweight: the *content* retained is good mutation fodder — 27 fast-lane findings this run
(sig_mismatch/we_error/timeout), classes that a type/spec-diverse accept corpus feeds. Fix the
mechanism, keep the fodder.

### I.1 Bounded behavior features (replaces string seen-sets)

Coarsen first, then hash into a fixed-size map. Per engine: a 2^20-bit virgin bitmap. Feature =
maphash of the coarse tuple → bit index; retain iff the bit was unset. Delete `behaviorSeen`,
`rawSeen`, `behaviorKeyCap`, `rawBehaviorKeyCap`, `seenSetFull` and the second key build.
Collisions lose a retention (AFL's bargain); saturation is observable (I.5).

Coarse tuple, replacing `BehaviorKey`'s field definitions:

- **Error class**: harden `NormalizeError` — mask single-quoted and backticked spans alongside
  double-quoted, `at line \d+`, and all remaining digit runs. Applies to `d.Error` (the proven
  leak) and `ourErr` alike. Crash-bucketing rule: over-mask, never under-mask.
- **Accept shape**: type *family* (drop params/digits: `numeric(10,2)` → `numeric`), spec-kind
  multiset with count bucketed {0,1,2,3+} instead of ordered spec list, statement-kind list,
  qualification arity. Built **from the structured digest** (`digest.Digest` is already
  unmarshaled) and from our canonical sig — no signature string re-parse on the oracle side.
- engine, `MaskSQLMode` unchanged.

Cost target ≤1µs/exec: one streaming maphash over the coarse fields, no intermediate strings.
The concrete-precision diversity this stops *retaining* already saturates the existing pool and
keeps being regenerated by mutation; it stops being a reason to grow the corpus, not a thing
the fuzzer stops trying. Validation: re-run the keyspace probe against the new key — expect
≤ ~10K distinct classes over the same 500K-row sample.

### I.2 Persist virgin state across restarts

Write the behavior bitmaps to `state/` (atomic tmp+rename, on the existing heartbeat) and load
at start — same lifecycle as the sancov accumulator. Kills the per-restart retention storm and
makes "quiet" durable.

### I.3 Per-case oracle edge attribution (protocol bump, no backcompat)

Plan 23 kept window-level attribution and its own caveat ("guesswork"). Do it exactly: the
oracle keeps a virgin sancov map and replies per case with a new-edge count. Retention =
count > 0, smallest-first cap stays.

Clean bump: the client requires the new reply shape, and the `covWindow`/`pollEvery` window
path is **deleted**, not kept as a fallback. Development runs against worktree-rebuilt
oracles: copy the server build trees (`build/mysql-build`, `build/mariadb-build`,
mtime-preserving so ninja stays incremental — the servers no-op, only driver.cc recompiles
and relinks) from the main checkout into the worktree's `build/`, rebuild via
`oracle/*/build.sh` (writes the manifest), re-run golden. Landing still rebuilds in
`staged/` so the manifest hash matches (merge-staged contract).

### I.4 Corpus hygiene v2

- `ddlfuzz corpus-distill` (offline): rewrite the **noise tier only** — keep the smallest row
  per coarse feature (I.1 key), drop the rest (measured: 418K rows ≈ 850 real classes).
  Behavior tier untouched — it's plausibly driving the current finding rate; distilling it
  needs an A/B (fixed-duration campaign, distilled vs full, findings/hour), not a size
  argument.
- Enforce the corpus byte budget for real: on `Add` over budget, evict noise-signal rows first,
  then largest-per-feature duplicates. `retention_skipped` starts meaning something.

### I.5 Stats (additive)

`behavior_bits` set-count + saturation % per engine, `retained_per_min`, and an instantaneous
`execs_per_sec_window` beside the lifetime average that hid this regression.

## Part II — e2e-gen dedup: one grammar, one fresh-name allocator

Successor to plan 22 (which built the table-driven grammar) and fix-e2e-rejects (which patched
the e2e lane's *copy* of the grammar). Goal: the constrained (e2e) profile rides the same
emission tables as the fast lane, and the fresh-name concept has one implementation. Plus the
two constrained-coverage openings the dedup makes nearly free: the exec-safe table constructs
the hand-rolled copy never learned, and the real session mode in the constrained `Ctx` so
mode-gated identifier spellings flow through the live plumbing checks.

### Diagnosis (2026-07-04)

Plan 22 rebuilt `Generate` as data (`tables.go`: 85 checklist constructs, weighted entries with
engine/mode gates) but left `GenerateConstrained` on the pre-22 hand-rolled path.
fix-e2e-rejects then had to fix exec-rejects in that copy only, growing the divergence:

- **Parallel grammar.** `genAlterSpec` (`gen.go:205`) is a 12-way switch re-spelling ~15 of the
  `alterSpecs` entries; `genAttrsSimple` (`gen.go:297`) re-spells a constrained subset of
  `columnAttrs`; the head/spec assembly in `GenerateConstrained` (`gen.go:72`) re-spells
  `genStatement` (`gen.go:135`) with hardcoded head probabilities. Drift is already live: the
  table lane got the plan-22 grammar and never got the fix-e2e-rejects allocator; the
  constrained lane got the allocator and none of the plan-22 grammar. Every future fix picks a
  side.
- **Half-shared is the worst of both.** Constrained `genRename` (`gen.go:194`) *does* fall
  through to the shared `renameForms` table — where `rename.multi_pair`,
  `rename.schema_qualified`, and maria `rename.wait_nowait` are guaranteed e2e exec-rejects
  (tables don't exist), and `rename.tables` is a guaranteed 1064 on mysql. ~45% of the 5%
  rename slice rejects by construction — exactly the class fix-e2e-rejects was written to
  kill, leaking through the one shared table because entries carry no constrained gate.
- **Three fresh-name allocators**, same concept (palette-preferred, live-set-filtered,
  synthesize on exhaustion), three spellings: `gen.(*Ctx).freshIdent` (`gen.go:268`, rng pick,
  `nf%d_%d`), `e2e.freshAlloc` (`rewrite.go:104`, deterministic pop, `nf%d_r%d`),
  `e2e.fallbackFreshName` (`fixture.go:233`, seq-keyed, `n%d_%d`).
- Minor doubles, disposition in II.3: `e2e.baseTypeVocab*` (`fixture.go:82`) vs
  `gen.typeEntries`; `e2e.ansiQuoteIdent`/`quoteIdent` vs `quoteMaybe`'s quoting arms;
  `Vocab.IsMariaDB` shadowing `Ctx.IsMariaDB`.

Why the lanes differ at all (binding on the design): the fast lane's oracle is parse-only, so
hostile/invalid syntax is free signal; the e2e lane diffs information_schema before/after a
*live execution*, so a statement that exec-rejects buys nothing. Constrained emission must stay
executable-by-construction against the fixture vocabulary; invalid and mode-*mismatched*
hostility belongs to corpus-rewrite and the fast lane. Mode-*matched* spellings (ANSI-quoted
identifiers under ANSI_QUOTES, brackets under MSSQL) are exec-safe and in scope (II.4).

### II.1 Constrained lane rides the tables

- `entry` gains one positional field: `CE bool` — constrained-eligible. Positional struct
  literals (existing style) force every current and future entry to state a disposition; no
  default. Weighting reuses `W`: no separate exec-lane distribution to maintain — the old
  12-way switch's frequencies were an artifact, nothing in the lane keys off construct
  frequency (no coverage feedback; throughput 20–50× oversized per 30 §9), and `W` among the
  eligible entries is ADD-heavy in roughly the same shape anyway. Fixture churn (add/drop
  balance → reset rate) is absorbed by `shouldReset` and watched via step 2G.
- `Ctx` gains `P *Profile` (nil = fast lane); `c.constrained()` ≡ `c.P != nil`. `entryOK`
  rejects `!e.CE` when constrained and additionally applies the profile's ID-level gates:
  `NoAlterRenameTo` → `spec.rename_table`, `NoConvertCharset` → `spec.convert_charset` (the
  flags keep their 30 §Interfaces meaning; they move from switch branches to gates).
- **One emission function per construct.** Where the hostile emission isn't exec-safe, gate the
  hostile sub-picks in place with `c.constrained()` rather than adding a parallel `CEmit` —
  divergence points stay inside the single Emit, greppable and reviewable. Concretely:
  - fresh-name positions (`spec.add_column`, `spec.add_column_list`, `spec.change_column`,
    `spec.rename_column`, `spec.rename_table`) call `c.freshIdent()` unconditionally — safe
    for the fast lane too (palette-preferred keeps the hostile spellings; the oracle is
    parse-only, dup-name pressure is not parse signal).
  - `genAttrs` → `genAttrs(c, typ)`. Constrained: exactly one draw over the `CE` entries,
    reproducing today's `genAttrsSimple` set — `attr.nullability` (no `NOT NULL ENABLE`),
    `attr.default` (type-gated: `hasAnyPrefix` families for `DEFAULT 'x,y'` / `DEFAULT 1`,
    else `DEFAULT NULL` only), `attr.comment` (`'ddlfuzz'` only), `attr.visibility`
    (`INVISIBLE`; `VISIBLE` mysql-only), plus the empty draw. Fast lane unchanged (0–4 attrs,
    full zoo, typ ignored).
  - `spec.add_index` constrained: drop `WITH PARSER ngram` (FULLTEXT-only), drop index
    `VISIBLE`/`INVISIBLE` on maria (no index visibility), keep name from `bareIdentifier` +
    live column. `spec.table_options` constrained: `ENGINE=InnoDB` only (today's
    default-branch behavior; `TABLESPACE ts`/`COMPRESSION` are exec-rejects).
  - Exec-safe sub-picks the hand-rolled copy lacked stay live in constrained mode — no gate:
    maria `maybeMaria("IF EXISTS ")` on MODIFY/CHANGE/DROP (live snapshot column → succeeds),
    `position()` on MODIFY/CHANGE (`AFTER <live col>`/`FIRST`), `RESTRICT`/`CASCADE` on DROP
    (parsed and ignored).
- **CE set** = every entry whose emission (with the gates above) is exec-safe against the live
  fixture: `spec.{add_column, add_column_if_not_exists, add_column_list, modify_column,
  change_column, drop_column, rename_column, add_index, add_check, keys_toggle, order_by,
  rename_table, convert_charset, algorithm, lock, force, table_options}`; heads
  `head.{alter_table, alter_online, alter_ignore, if_exists, wait_nowait}` (the last two
  maria-only, target the existing fixture, and `wait_nowait`'s `tableSuffix` is honored by the
  shared assembly); `rename.table` only (kills the multi_pair/schema_qualified/tables reject
  leak, measurable in `e2e-exec-rejects.jsonl`). Everything else false. This deliberately
  widens the constrained construct set beyond today's hand-rolled emissions (`IF [NOT] EXISTS`
  forms, `WAIT/NOWAIT`, positions on MODIFY/CHANGE) — each addition exec-safe by the argument
  above and pinned by the allowlist test; frequencies are `W`-proportional, not preserved.
- Assembly: extract `genOne(c) string` from `genStatement` (head via table honoring
  `c.tableSuffix`, spec loop, rename branch). `GenerateConstrained` = normalize vocab, build
  `Ctx{P: &p}`, spec count `1 + IntN(maxSpecs)`, call `genOne`; no benign heads, no `;`-chains,
  no SET wrap, no decorate, no NUL — all keyed off `c.constrained()` in `Generate`'s outer
  shell, which becomes the only caller of those stages. Delete `genAlterSpec`,
  `genAttrsSimple`, and the constrained head literals.

### II.2 One fresh-name allocator

`internal/gen`:

```go
type FreshPool struct{ ... }
func NewFreshPool(palette, live []string) *FreshPool
func (p *FreshPool) Next(r *rand.Rand) string // r==nil: first-free (deterministic); else uniform over free
```

Synthesis on exhaustion: `nf<seq>_<n>` where `n` = `r.Uint64N(1_000_000)` or `seq` when
deterministic; always checked against live+used. `Ctx.freshIdent` becomes a lazy
`NewFreshPool(V.FreshNames, V.Columns)` + `Next(c.R)`; `e2e.newFreshAlloc`/`freshAlloc`
deleted, `rewriteCorpusStatement` uses `gen.NewFreshPool(FreshNames, <live keys>)` +
`Next(nil)` — order-preserving first-free keeps `rewrite_fallback_test.go` determinism (update
only synthesis spellings if asserted).

`fallbackFreshName` stays: its `FreshNames[seq%16]` selection is a smoke-lane contract
(palette indices 13/14 must be reachable by seq, `rewrite_fallback_test.go` asserts it) and it
is stateless per call — a pool would fake the seq keying.

### II.3 Dispositions for the minor doubles

- `baseTypeVocab*` (fixture.go): **keep in e2e.** Fixture-domain data (executable written
  forms matching the fixture's column families), already parameterized through `Vocab.Types`;
  deriving it from `typeEntries` would couple e2e exec-safety to fast-lane hostility.
- `ansiQuoteIdent`/`quoteIdent` (e2e): keep — trivial, and e2e quoting is policy
  (always-quote) not fuzz (quote-maybe).
- `Vocab.IsMariaDB`: fold reads onto `Ctx.IsMariaDB` (set once in the two constructors); keep
  the Vocab field for 30 §Interfaces compatibility.

### II.4 Real session mode in the constrained Ctx

Today `GenerateConstrained` builds `Ctx{Mode: 0}`, so `quoteMaybe`'s ANSI/bracket arms
(`gen.go:344-347`) are dead in e2e — generated DDL never carries mode-gated identifier
spellings through the live pipeline, though the mode-gated lexer is where the confirmed bug
class lives (P5/P6) and the compare side is already mode-correct (`expectRelevant` is
readback-derived).

- `Profile` gains `Mode uint64` (additive field; contract-addition note against 30
  §Interfaces). `GenerateConstrained` sets `ctx.Mode = p.Mode`.
- Worker: `relevant` (the readback-derived bitmask, `worker.go:146`) threads through
  `ws.statement` → `ws.generateConstrained` → `Profile.Mode`. Two unexported signatures.
- Exec-safe by construction: `quoteMaybe` emits a spelling only when its mode bit is set, and
  the bit comes from what the session actually has. Single-quoted literals unaffected by
  ANSI_QUOTES; decorate/respellString never run constrained, so NO_BACKSLASH_ESCAPES stays
  inert. Net effect is identifier spelling only.

## Sequencing (binding)

The parts are code-independent (disjoint files; the only shared package, `internal/corpus`,
is written by Part I and only read by Part II via `RandomSQL`, `e2e/rewrite.go:20`). Two
interactions force order and timing:

1. **Watch-window integrity.** Part II step 2G watches e2e reject rate and fast-lane
   retention/coverage as no-regression canaries; Part I replaces the retention mechanics
   (baseline jump) and `corpus-distill` shifts the corpus-rewrite sampling distribution the
   e2e lane draws from. So: land I.1+I.2 first (small Go-only diff; restores throughput and a
   live signal, making every later watch window 3–4× more sensitive), re-baseline, then land
   Part II, then I.3; **run `corpus-distill` only after Part II's watch window closes.**
2. **Oracle rebuild travels with 1C.** I.3 bumps the wire protocol with no backcompat, so it
   lands atomically with rebuilt binaries: dev-rebuild in the worktree (copied build trees),
   landing rebuild in `staged/` (manifest + golden) per the merge-staged contract. Gates
   before 1C run against the pinned binaries; from 1C on, against the worktree-rebuilt ones.

## Implementation steps

All in `worktrees/feedback-e2e`, inside `tools/ddlfuzz/` (plus `e2e/`). Gate after every step:
`gofmt -l . && go build ./... && go vet ./... && go test ./...`. If the module fails on missing
generated protos or oracle binaries, copy `flow/generated/` / `build/oracle-*` + manifest from
the main checkout (read-only source; never write to the main checkout or `state/`).

- **1A — coarse features + bitmap** (I.1): `NormalizeError` hardening (table-driven tests from
  real server error text); coarse-tuple builder from digest/result; bitmap in `engineState`;
  swap `retainBehavior` internals; delete the string sets. Validate: keyspace probe re-run,
  ≤ ~10K classes / 500K rows.
- **1B — persistence** (I.2): bitmap save/load + heartbeat write; restart test on a fixture
  state dir.
- **1C — per-case edges** (I.3): copy the server build trees from the main checkout
  (mtime-preserving); driver.cc virgin map + per-case counts in both oracles; client parse +
  retention switch; delete `covWindow`/`pollEvery`; rebuild via `oracle/*/build.sh`; golden +
  gate against the rebuilt binaries.
- **1D — corpus-distill + budget eviction** (I.4): fixture-DB tests for tier rewrite and
  eviction order. Command lands; **execution deferred per §Sequencing**.
- **2A — FreshPool** (II.2): type + tests (palette-preferred, live-filtered, no repeats,
  deterministic when r==nil, synthesis distinct); switch `Ctx.freshIdent` and `e2e/rewrite.go`
  onto it; delete `freshAlloc`. `gen_fresh_test.go` and `rewrite_fallback_test.go` green
  (spelling-only updates allowed).
- **2B — CE plumbing** (II.1): `CE` on `entry` (every literal updated — disposition forced),
  `Ctx.P`, constrained-aware `entryOK` with profile ID-gates. No caller change yet.
- **2C — emission unification** (II.1): `genAttrs(c, typ)` with constrained arm; constrained
  gates inside `spec.add_index`/`spec.table_options`/`attr.*`; fresh-name positions →
  `freshIdent`. Fast-lane byte-safety fuzz test still green.
- **2D — assembly swap** (II.1): extract `genOne`; `GenerateConstrained` → tables; delete
  `genAlterSpec` + `genAttrsSimple` + hand-rolled head/rename literals. Port fix-e2e-rejects
  invariants (gen_fresh_test) to the table path.
- **2E — session mode** (II.4): `Profile.Mode`; thread `relevant` through
  `statement`/`generateConstrained` in `e2e/worker.go`; `ctx.Mode = p.Mode`.
- **2F — tests** (II.1/II.4):
  - Shape-set: 50k constrained statements per engine (across the mode palette) emit only CE
    construct shapes and every CE construct appears (presence, not frequency).
  - Exec-safety audit per CE entry: force-emit N times constrained; assert no dup fresh names
    (incl. `IF NOT EXISTS` form), no `VISIBLE` column attr on maria, defaults type-gated, no
    `WITH PARSER`, no maria index visibility, no `TABLESPACE`/`COMPRESSION`,
    `WAIT/NOWAIT`/`IF EXISTS` heads maria-only, renames only `fixture → <fresh>`.
  - Mode audit: over N constrained draws per mode — `"…"` identifiers only when the
    ANSI_QUOTES bit is set, `[…]` only when maria and the MSSQL bit is set, backticks always
    legal; `Mode=0` output backtick/bare only.
  - Checklist test extended: CE set == a named allowlist so future widening is a conscious
    diff.
- **3 — verify + land** (per §Sequencing): full gate; `replay --all` green; roll onto
  `ddlfuzz-staged` in the landing order above; `ddlsuper merge-staged` (backgrounded) per
  landing; oracle rebuild in staged for the 1C landing. Watch per landing:
  `execs_per_sec_window`, `retained_per_min`, bitmap saturation (Part I); e2e reject rate ≤
  pre-land, ORACLE/MSSQL palette slice specifically (Part II).

## Constraints (binding)

- `gen.GenerateConstrained(r, v, p)` signature unchanged; existing `Vocab`/`Profile` fields
  and semantics unchanged; `Profile.Mode` is the one additive extension. `e2e/worker.go`
  changes limited to threading `relevant` into `statement`/`generateConstrained`.
- Constrained construct-set changes are exactly: the rename-forms fix, the CE additions
  enumerated in II.1, and mode-gated identifier spellings (II.4) — pinned by allowlist and
  audit tests. Nothing else.
- stats.json additive (D12); corpus.db schema additive, old rows readable; `signal` column
  semantics unchanged.
- Wire protocol bump only in 1C, no backcompat — lands atomically with rebuilt oracles;
  worktree gates from 1C on run against worktree-rebuilt binaries; staged rebuild at landing
  with manifest + golden.
- Byte-safety invariant everywhere; sentinel never emitted. `-mut-ratio` semantics unchanged.
- Campaign live: land via merge-staged only; nothing committed in the main checkout.

## Dropped

- Distilling the behavior tier (pending A/B) — 22GB is disk, not heap; startup already
  tier-samples noise.
- Favored-entry scheduling (AFL `top_rated`) for base picks — revisit when retention is
  healthy and `es.bases` composition is measurable.
- Any per-exec string key or exact seen-set — the probe shows exactness buys collision-freedom
  on a key that shouldn't be injective in the first place.
- Deriving `baseTypeVocab*` from `typeEntries`; per-construct accept telemetry; bandit grammar
  weights; exact edge bisect; external corpora — unchanged from the source drafts.

## Exit metrics

Part I, within a day of its landings: steady-state ≥800K execs/s with retention *flowing*
(nonzero `retained_per_min`, tapering); bitmap saturation well under 50%; a fuzzer restart
resumes without a retention storm; noise tier within budget post-distill. Finding rate at
worst unchanged from the ~250K/s regime — at 3–4× the execs, anything else means the coarse
key over-collapsed and the I.4 A/B machinery tells us where.

Part II: `genAlterSpec`/`genAttrsSimple`/`freshAlloc` deleted; one grammar walk in the
package; e2e reject rate not regressed (rename slice gone) with `IF [NOT]
EXISTS`/`WAIT`/positioned-MODIFY forms and ANSI/bracket identifier spellings observable in
`e2e-live-accepted.jsonl`; next constrained-lane fix is a table edit, not a second copy.
